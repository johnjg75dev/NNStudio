📚 Architecture Zoo

MLP — Multi-Layer Perceptron

The foundational feedforward network. Every neuron connects to every neuron in the next layer (fully connected). Trains via backpropagation and gradient descent. Best for tabular data, logic gates, function approximation. This is what runs in the Train tab.

CNN — Convolutional Neural Network

Uses sliding kernel filters to detect local spatial patterns (edges, textures). Parameters are shared across positions — very efficient for image data. Key layers: Conv → ReLU → MaxPool → Dense. Foundation of ResNet, VGG, EfficientNet.

Autoencoder

Hourglass architecture: Encoder compresses input to a small latent vector (bottleneck), Decoder reconstructs the original. Trained to minimise reconstruction error. The bottleneck forces compact representations. Used for compression, denoising, anomaly detection. Core component of VAEs and Stable Diffusion.

RNN / LSTM / GRU

Recurrent networks process sequences by maintaining a hidden state across timesteps. Vanilla RNNs suffer from vanishing gradients. LSTMs add forget/input/output gates to selectively remember and forget. GRUs are a simplified variant. Best for time series, language modelling, speech.

Transformer

Replaces recurrence with Self-Attention: every token attends to every other token in parallel. Multi-head attention learns different relationships simultaneously. Residual connections + LayerNorm enable very deep stacks. Foundation of GPT, BERT, T5 — virtually all modern AI.

ViT — Vision Transformer

Applies a standard Transformer to images by splitting into fixed-size patches (e.g. 16×16), linearly embedding each patch as a token, prepending a [CLS] token, then running through a Transformer encoder. [CLS] output used for classification. Outperforms CNNs at scale (Dosovitskiy et al., 2020).

VAE — Variational Autoencoder

Like an autoencoder, but the encoder outputs a distribution (μ, σ²) rather than a single point. Sampling uses the reparameterisation trick: z = μ + σ·ε. Trained with reconstruction loss + KL divergence. Enables smooth latent-space interpolation and generation.

Diffusion / Stable Diffusion

Forward: gradually add Gaussian noise over T timesteps. Reverse: a U-Net learns to predict and remove noise at each step. At inference: start from pure noise and iteratively denoise. Stable Diffusion adds a VAE (works in latent space) and CLIP text conditioning via cross-attention.

GAN — Generative Adversarial Network

Generator G(z) maps noise to fake data. Discriminator D(x) distinguishes real from fake. Adversarial training: G tries to fool D; D improves, forcing G to improve. Produces very sharp outputs but training is notoriously unstable (mode collapse, gradient vanishing).

⚙️ Hyperparameter Guide

Learning Rate — The Most Critical Hyperparameter

Too high → loss diverges or oscillates wildly. Too low → extremely slow convergence. Adam is much less sensitive to LR than SGD. Rule of thumb: start at 1e-3 with Adam. Use learning rate finders or cosine annealing schedules in production.

Batch Size and Gradient Quality

Large batch: smoother gradients, faster GPU use, but often worse generalisation ("sharp minima"). Small batch: noisy gradients act as implicit regularisation. Linear scaling rule: if you double batch size, also double LR. Full-batch (all data at once) is used here.

Overfitting vs Underfitting

Underfitting: Model too simple or not trained enough. High training loss. Fix: more capacity, lower regularisation, train longer.
Overfitting: Low training loss, high validation loss. Fix: Dropout, weight decay, data augmentation, early stopping.

Vanishing & Exploding Gradients

In deep networks, gradients can exponentially shrink (vanish) or grow (explode) as they propagate backwards. Solutions: residual/skip connections, batch/layer normalisation, gradient clipping, He/Xavier initialisation, LSTM gates, modern activations (GELU, Swish).

Weight Initialisation

Bad init = symmetry problem or dead start. Xavier/Glorot: scale ~ √(1/fanIn), for Tanh/Sigmoid. He init: scale ~ √(2/fanIn), for ReLU. Zero init: always broken — all neurons learn identically (symmetry not broken). This trainer uses He init.

Backpropagation — How Networks Learn

1. Forward pass: compute output. 2. Compute loss. 3. Backward pass: apply chain rule layer-by-layer to compute ∂Loss/∂W for every weight. 4. Optimizer applies gradients to update weights. Repeat millions of times. Differentiable activations are essential for step 3.

Self-Attention Mechanism

For each token, compute Query (Q), Key (K), Value (V) via learned linear projections. Attention scores = Softmax(QKᵀ / √d_k). Output = scores · V. Every position can "look at" every other in O(n²) time. Multi-head attention runs h parallel heads and concatenates, learning different relationship types simultaneously.

Dropout & Regularisation

Dropout randomly zeros neurons during training (disabled at inference), forcing the network to learn redundant representations. Weight decay (L2) penalises large weights. Together they prevent the network from memorising training data. AdamW's decoupled weight decay is more principled than L2 in Adam.
